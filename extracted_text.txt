Grove — Help Feature
Feature Spec · For Claude Code Implementation
Prepared March 2026 · Sprout Internal

1. Overview
Help is a persistent feature that allows any logged-in Grove user to submit bug reports or ask questions about the app. All submissions are visible to everyone. Gardeners (admins) can mark items as resolved or answered.

Feature name
Help
Entry point
Floating button, fixed bottom-right, visible on all pages
Item types
Report (bug) · Ask (question)
Visibility
All submissions visible to all logged-in users
Resolve/Answer
Gardeners only
Notifications
None — not in scope for v1
Scope
General only — not linked to specific Plants or Seeds
Auth required
Yes — logged-in users only, submitter name auto-filled from session

2. Entry Point — Floating Button
A floating action button (FAB) sits in the fixed bottom-right corner of the app, always visible regardless of which tab or view the user is on. Clicking it opens the Help slide-over panel from the right. The panel opens without navigating away from the current page.

5. Submit View
Opens when user clicks + Report or + Ask. Replaces the feed content within the same panel (feed header remains, filter tabs and action row are hidden during submission). Clicking back returns to the feed.

Submit View Elements

Back button — arrow icon + &quot;Back to Help&quot;, 12px mushroom-500, top of view
View heading — &quot;Submit a report&quot; or &quot;Ask a question&quot; — 14px, font-weight 600
Type toggle — Report / Ask toggle, pre-selected based on which button triggered open
  Report selected: tomato-100 bg, tomato-700 text, #FFCCBC border
  Ask selected: blueberry-100 bg, blueberry-700 text, #BBDEFB border
Submitter strip — avatar initials + &quot;Submitting as [Name]&quot; — mushroom-50 bg, border mushroom-200
Title field — required, placeholder: &quot;Brief description...&quot;
Description field — optional, textarea, placeholder: &quot;Steps to reproduce, or more context...&quot;
Submit button — full width, kangkong-700 bg, white text, hover: kangkong-800

Form Validation
Title is required — Submit button does nothing if title is empty
Description is optional — can submit without it
Submitter name is auto-filled from session — not shown as an editable field

On Submit
New help_item record created in DB
Panel returns to feed view automatically
Filter resets to All, page resets to 1
New item appears at top of feed

6. Editing Items
Submitters can edit their own Report or Ask after posting
Edit is available only while status is open or unanswered — locked once Gardener resolves/answers
Editing opens the same submit form pre-filled with existing title and description
Submitter, type, and created_at are not editable

7. Gardener Actions
Gardeners (is_gardener = true) have additional controls visible on each feed item. These are not visible to regular employees.

Action
Trigger
Result
Mark as Resolved
Button on Report item (Gardener only)
status → resolved, resolved_by and resolved_at recorded, item opacity 0.5
Mark as Answered
Button on Ask item (Gardener only)
status → answered, resolved_by and resolved_at recorded, item opacity 0.5
Delete item
Delete button on any item (Gardener only)
Hard delete from DB, item removed from feed immediately

No Re-open in v1
Once marked resolved or answered, there is no re-open action in v1. Gardeners can delete and submitter can repost if needed.

11. Hard Constraints
No localStorage or sessionStorage — Supabase session only
No notifications in v1 — no email or in-app alerts on submission or resolution
No threading or comments — each item is a standalone record
No linking to Plants or Seeds — general scope only
No re-open action — once resolved/answered, status is final in v1
Submitter name is always auto-filled — never a manual input
Upvoting own item is not allowed — enforced in both UI and RLS
Permission checks on all mutations enforced in both UI and Supabase RLS

12. Open Items
Not in scope for v1 — revisit after launch
Re-open action: allow Gardeners to revert resolved/answered status
Gardener response: allow Gardeners to add a public reply to an Ask
Notification: in-app or email ping when item is resolved/answered
Admin view: dedicated Gardener dashboard showing open items count
Search: text search within the Help feed
